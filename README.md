## Overview

A machine learning system that forecasts weekly paddy-stubble fire activity across Punjab at a **7 km grid resolution**, one week ahead. The model integrates satellite fire detections, vegetation indices, climate reanalysis, and Punjab policy variables to predict fire-weighted counts at each grid cell.

Punjab burns approximately 19 million tonnes of paddy stubble annually during October–November. The resulting smoke contributes over 40% of Delhi's peak-season PM2.5. Current monitoring is reactive. This system enables district officers to pre-position enforcement and crop-residue management (CRM) resources before burning peaks.

## Data sources

| Source | Resolution | Variables | Range |
|---|---|---|---|
| NASA FIRMS | 1 km / daily | Fire detections, FRP, brightness | 2018–2023 |
| MODIS MOD13Q1 | 250 m / 16-day | NDVI, EVI | 2018–2023 |
| ERA5-Land | 0.1 deg / daily | Temperature, soil moisture, wind, VPD | 2018–2023 |
| Punjab Policy | Yearly | 8 variables (super-seeder, NGT, ex-gratia, MSP, CRM funds) | 2018–2023 |

---

## Feature engineering

- **Grid:** 7 km x 7 km cells (0.07 deg), 1,040 cells over Punjab (29.7–32.5°N, 74.0–76.5°E)
- **FIRMS lag features:** fire counts last week, same week last year, 3-year average, neighbor cell counts, intensity lags (FRP, brightness, night-fire fraction)
- **NDVI:** 16-day MOD13Q1 composites mapped to ISO weeks via KD-tree; baseline computed from train years only; anomaly = current minus train baseline
- **Weather:** ERA5-Land daily aggregated to weekly (mean/max/min/std); NaN gaps (~47% of cells) filled by per-(year, week) KD-tree nearest-neighbour interpolation
- **Policy:** 8 year-level variables hand-coded from PIB press releases and CEEW reports

## Models trained

| Model | PR-AUC (test 2023) | MAE |
|---|---|---|
| Persistence baseline | 0.750 | 2.57 |
| Logistic Regression | 0.9 | — |
| Random Forest | 0.889 | 2.87|
| XGBoost-Tweedie (tuned) | **0.869** | **1.95** |
| ConvLSTM (spatial) | — | 0.43 |

XGBoost-Tweedie with `tweedie_variance_power=1.5` handles zero-inflated count data. Tuned with Optuna (100 TPE trials, objective: PR-AUC on val 2022). ConvLSTM motivated by Moran's I = 0.54 on XGBoost residuals.

## Leakage audit — 8 checks, all passed

1. Cross-year boundary clean (no week-48 of 2022 leaking into week-40 of 2023)
2. Intensity features properly lagged (FRP, brightness, night-fire fraction at t-1)
3. Cartesian join sanity (1,040 x 6 x 9 = 56,160 rows, no duplicates)
4. NDVI baseline from train years only
5. Spatial overlap acknowledged (same grids in train/test — only temporal generalisation claimed)
6. Ablation behaviour sensible (NDVI-only ~0.55, history-only ~0.88)
7. Shuffled-target control returned PR-AUC 0.45 (base rate)
8. No grid_id passed directly as a feature

Initial v1 model showed PR-AUC 0.997 — traced to NDVI NaN values aligning perfectly with zero-fire rows (merge artifact). Rebuilt from raw MOD13Q1 rasters. Honest post-fix: **0.869**.

## Running the dashboard

```bash
# 1. Install dependencies
pip install streamlit pandas numpy plotly xgboost shap pyarrow

# 2. Pre-compute dashboard data (one-time, ~60 seconds)
python precompute.py

# 3. Launch
streamlit run app.py
# or: /opt/anaconda3/bin/streamlit run app.py
```

Open http://localhost:8501. The dashboard has 5 tabs:
- **Overview** — hero map, headline metrics, project narrative
- **Historical Replay** — week-by-week 2023 scrubber with SHAP cell inspector
- **Forecast Simulator** — filter by district, week, confidence threshold
- **Model Explorer** — SHAP, ablation, lead-time decay, Moran's I, CPCB validation, counterfactuals
- **Methodology** — full pipeline, data sources, leakage audit, benchmark comparison
